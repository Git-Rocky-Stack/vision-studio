## Completed Work

The workflow graph redesign is now implemented end to end for the first usable surface:

- Workflow records include `graph` data with typed nodes, edges, literal inputs, and linked inputs.
- Default workflow seed data now includes a ComfyUI-style prompt graph.
- Store actions support graph editing:
  - add workflow node
  - move workflow node
  - connect workflow nodes
  - delete workflow node and connected edges
  - delete workflow edge
- The Workflow workbench center surface renders an editable graph editor instead of the previous run-plan placeholder.
- Graph nodes and edges are selectable.
- Nodes can be added from toolbar actions.
- Nodes can be connected through the selected-source connection flow.
- Nodes can be dragged and preview their movement live while dragging.
- Selected nodes and edges can be deleted.
- The inspector reflects selected node/edge details.
- The workbench can export the active graph as ComfyUI API JSON.
- The export panel resets when the active workflow or graph changes.

The ComfyUI exporter is in place:

- `src/features/workflow/comfyExport.ts` converts internal graph records to ComfyUI API prompt JSON.
- Literal inputs are emitted directly.
- Linked inputs become `[nodeId, outputName]` tuples.
- Export validates missing source nodes and duplicate target input collisions through tests.

Review-hardening fixes from the final workflow graph pass are included:

- `CheckpointLoaderSimple -> KSampler` now connects `MODEL` to `model`, not `positive`.
- `CLIPTextEncode -> KSampler` still connects `CONDITIONING` to `positive`.
- `KSampler -> PreviewImage` and `KSampler -> SaveImage` connect `IMAGE` to `images`.
- Delete clears local selection so repeated delete cannot target a stale node.
- Drag updates node/edge preview positions during pointer move, not only on pointer release.
- ComfyUI JSON export does not remain stale after switching workflows.

The Invoke-style workbench layout remains in place:

- Generate and Quick use a left settings dock, center Canvas/Viewer/Workflow mini-tabs, and right Boards/Gallery context stack.
- Edit uses a left inspector, tool rail, center work surface, and right Layers/Gallery stack.
- Viewer can open recent Gallery output, branch a result back into Generate, pin comparisons, and add an output to the active board.
- Boards can be created from the context dock, opened in Storyboard, populated from Viewer output, grouped by scene status, and annotated with context metadata.
- Narrow workbench widths preserve the center surface instead of letting side docks crush it.

The semantic typography/glyph guard work remains active:

- Decorative text glyphs and HTML glyph entities are blocked by `src/styles/ui-glyphs.test.ts`.
- The semantic typography guard covers the main shell, canvas comparison/queue/progress surfaces, Generate/Quick/Storyboard panels, several Storyboard components, Region Lock edit surfaces, and core UI fallback/dialog/error components.
- `src/index.css` includes semantic typography helpers: `type-title`, `type-section`, `type-ui`, `type-caption`, `type-badge`, and `type-meta`.

## Current Workflow Graph Capability

The current implementation is a functional first pass, not yet a full ComfyUI clone:

- Nodes are rendered as lightweight absolute-position buttons.
- Edges are SVG paths with small selectable midpoint controls.
- Connection creation is toolbar-driven:
  1. Select source node.
  2. Click `Start connection from selected node`.
  3. Click target node.
- Input/output slot selection is currently heuristic based on class type and source output.
- Node input editing is not yet exposed in the inspector.
- Node type schema/port metadata is not yet centralized.
- There is no import path for ComfyUI JSON yet.
- There is no run execution bridge to a backend graph runner yet.

## Suggested Next Implementation Work

1. Add editable node input controls in the graph inspector.

   The graph already stores literal inputs and linked inputs. The next useful feature is editing common literals such as prompt text, checkpoint name, sampler seed, steps, CFG, and filename prefix from the selected-node inspector.

2. Centralize node definitions and port metadata.

   Move class-specific labels, default inputs, output slots, target slot rules, and default positions into a shared workflow node registry. This will reduce hard-coded logic in `WorkflowGraphEditor.tsx`, `WorkflowWorkbench.tsx`, and the exporter tests.

3. Add explicit port UI.

   Replace toolbar-only connection with source/output and target/input port selection. This is the next step toward real ComfyUI-style graph editing and will remove most slot heuristics.

4. Add ComfyUI API JSON import.

   The exporter exists. A paired importer should convert ComfyUI prompt JSON back into `WorkflowGraph` records with positions where available, generated positions where missing, and validation diagnostics for unsupported nodes.

5. Add graph validation diagnostics.

   Surface missing links, duplicate target inputs, missing required literals, unknown node classes, and disconnected output nodes in the workbench instead of only throwing during export.

6. Add graph execution integration.

   Decide whether the app will queue exported ComfyUI API JSON directly to a ComfyUI-compatible backend, translate it to the existing backend contract, or support both. Keep exporter tests as the contract boundary.

7. Improve graph editor interaction polish.

   Useful next items: marquee/pan/zoom, keyboard delete, undo/redo integration, snap-to-grid, edge labels, node type search, and better selected-node visual affordances.

8. Continue semantic typography cleanup in small guarded slices.

   Good next candidates are tested components with contained typography usage, for example `src/components/ui/KeyboardShortcuts.tsx`, `src/components/edit/ColorPicker.tsx`, `src/components/edit/CropControls.tsx`, or `src/components/edit/TextControls.tsx`.
